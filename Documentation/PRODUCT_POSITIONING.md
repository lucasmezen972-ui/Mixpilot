# Positionnement produit MixPilot

## Promesse

MixPilot aide à préparer et superviser un set DJ fiable, puis à gérer les enchaînements en sécurité avec djay Pro, rekordbox ou Serato DJ Pro.

## Un seul produit

MixPilot ne contient pas trois applications cachées derrière un sélecteur.

Le produit repose sur :

- un moteur de préparation ;
- un moteur de transitions ;
- un système de sécurité ;
- une application iPhone ;
- trois backends DJ interchangeables.

## Trois backends officiels

### djay Pro

Positionnement : Autopilote natif et contrôle avancé.

Modes prévus :

- Automix supervisé ;
- MixPilot avancé par commandes validées.

### rekordbox

Positionnement : Mode Performance et installations professionnelles.

Forces existantes :

- import XML et JSON ;
- mapping MIDI CSV validé ;
- versionnement et provenance des correctifs ;
- validation commande par commande.

### Serato DJ Pro

Positionnement : Contrôle MIDI avec configuration guidée.

Forces existantes :

- contrôleur virtuel CoreMIDI ;
- installation XML ;
- sauvegarde et restauration ;
- observation de l’interface ;
- parcours de validation.

## Même importance, capacités différentes

Les trois backends apparaissent au même niveau dans l’application et la documentation.

Leur niveau de compatibilité dépend toutefois :

- de la version installée ;
- du mapping ;
- des permissions ;
- du contrôleur ;
- des fonctions exposées ;
- de la validation réalisée sur le Mac cible.

MixPilot adapte le plan au lieu de prétendre que toutes les commandes sont disponibles partout.

## Parcours utilisateur

1. Choisir le logiciel DJ.
2. Vérifier la connexion.
3. Importer ou lire une playlist.
4. Préparer le set.
5. Tester une transition.
6. Vérifier le système.
7. Lancer le Live.
8. Superviser depuis l’iPhone.
9. Reprendre la main à tout moment.

## Local d’abord

Le Live reste local et ne dépend jamais :

- d’Internet ;
- des services en ligne ;
- de l’iPhone ;
- d’une IA ;
- d’un correctif distant non installé.

Une panne en ligne peut désactiver les diagnostics et mises à jour. Elle ne doit pas couper la musique ni empêcher le contrôle manuel.

## Honnêteté de validation

MixPilot distingue toujours :

- code et tests automatisés ;
- simulations ;
- validation du backend ;
- validation du matériel ;
- fonction bloquée par la plateforme ;
- échec réel.

Aucune simulation n’est présentée comme une réussite matérielle.
